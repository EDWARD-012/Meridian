Design 1 — Full Entity Relationship Schema

Includes explicit Cart → CartItem → Product (cart values) and Order → OrderTracking (track order timeline).

erDiagram
    USER ||--o| CART : "owns (1:1)"
    USER ||--o{ ADDRESS : has

    CART ||--o{ CART_ITEM : "contains (cart values)"
    PRODUCT ||--o{ CART_ITEM : "referenced in cart"

    PRODUCT ||--o{ ORDER_ITEM : "sold in order"

    ORDER ||--o{ ORDER_ITEM : contains

    ORDER ||--o{ ORDER_TRACKING : "track order timeline"

    USER {
        int id PK
        string username UK
        string email UK
        string password_hash
    }

    USER_PROFILE {
        int id PK
        int user_id FK UK
        string phone
    }

    CART {
        int id PK
        int user_id FK UK
        datetime updated_at
    }

    CART_ITEM {
        int id PK
        int cart_id FK
        int product_id FK
        int quantity
        decimal line_total "computed: qty * price"
    }

    CATEGORY {
        int id PK
        string name UK
        string slug UK
    }

    PRODUCT {
        int id PK
        int category_id FK
        string name
        decimal price
        int stock_quantity
        string image_url
    }

    ADDRESS {
        int id PK
        int user_id FK
        string full_name
        string line1
        string city
        string pincode
    }

    ORDER {
        int id PK
        int user_id FK
        int address_id FK
        decimal total_amount
        enum status
        datetime order_date
    }

    ORDER_ITEM {
        int id PK
        int order_id FK
        int product_id FK
        int quantity
        decimal unit_price
        decimal subtotal
    }

    PAYMENT {
        int id PK
        int order_id FK UK
        string payment_method
        decimal amount
        enum status
        string transaction_id
    }

    ORDER_TRACKING {
        int id PK
        int order_id FK
        enum status
        string message
        string location
        datetime recorded_at
    }

Cart Value Calculation

ConceptSQL / Logic
Line totalcart_item.quantity × product.price
Cart subtotalSUM(line_total) for all cart_items in user's cart
ConstraintUNIQUE(cart_id, product_id) — one row per product per cart

Order Tracking Flow

StatusMeaning
pendingOrder placed
confirmedPayment confirmed / seller accepted
processingBeing packed
shippedLeft warehouse
out_for_deliveryWith delivery partner
deliveredCompleted
